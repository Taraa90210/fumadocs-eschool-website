# Leave History & Reports

Once a leave request is submitted, it is archived in the **Leave History** registry. This allows administrators to track the status of all institutional permit requests and maintain an accurate audit trail of absences.

---

### ðŸ“„ **Leave Registry Overview**

The registry table presents the following critical data points for each submission:

| Data Field | Administrative Context |
| :--- | :--- |
| **Sequence** | The institutional tracking number for the request. |
| **Start Date** | Commenced date of the leave. |
| **End Date** | Concluding date of the leave. |
| **Justification** | The provided reason for the absence. |
| **Attachments** | The number of supporting documents provided. |
| **Administrative Status** | Pending, Approved, or Rejected by the school administration. |
| **Actions** | Utilize the **Actions (â‹®)** menu to **View** or **Delete** specific entries. |

### âŒ **Managing Existing Submissions**

Should a modification to a submitted request be necessary:
*   Direct editing of submitted entries is **restricted** to ensure data integrity.
*   Administrators should **Delete** the incorrect entry via **Actions > Delete** and facilitate a new submission with updated information.
