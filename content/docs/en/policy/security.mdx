eSchool is committed to protecting the privacy and security of all users by implementing the highest security standards.

---

### Security Measures

#### 1. Data Encryption
- **Transmission**: All sensitive data is encrypted using SSL/TLS protocols during transmission.
- **Storage**: Data at rest is encrypted using industry standards (AES-256).
- **Database**: Database encryption to protect academic and personal data.

#### 2. Role-Based Access Control (RBAC)
- **Access Restriction**: Each user can only access data according to their role.
- **Permission Hierarchy**: Admins, Teachers, Students, and Parents have different access levels.
- **Audit Trail**: All access and data changes are recorded for auditing.

#### 3. Backup and Recovery
- **Automated Backup**: Data is backed up automatically every day.
- **Redundancy**: Backups are stored in multiple locations for maximum security.
- **Disaster Recovery**: Fast recovery procedures in the event of an incident.

#### 4. Monitoring and Audit
- **Real-time Monitoring**: System is monitored 24/7 to detect suspicious activities.
- **Security Audit**: Periodic security audits to identify vulnerabilities.
- **Penetration Testing**: Routine penetration tests to ensure system security.

---

### Security Practices for Users

#### For All Users
- **Strong Password**: Use a password of at least 8 characters with a combination of letters, numbers, and symbols.
- **Do Not Share**: Never share your password with anyone.
- **Logout**: Always logout after using the system.
- **Secure Device**: Use a secure and trusted device.

#### For Administrators
- **Two-Factor Authentication**: Enable 2FA for admin accounts.
- **Access Review**: Periodically review and update user access rights.
- **System Update**: Ensure the system is always updated with the latest security patches.

---

### Security Reporting

If you find a security vulnerability or suspicious activity:

1. **Report Immediately**: Contact the school IT team or eSchool support.
2. **Do Not Exploit**: Do not attempt to exploit the discovered vulnerability.
3. **Documentation**: Record the details of the findings to assist investigation.

**Security Contact**: security@eschool.ac.id
